# Quick Task 260401-lq1: Fix "all tier" Min Spend update

## Root Cause

The backend (`applyUpdateSurgical`) already handles "all tiers" correctly: when
`action.voucher_name` is falsy, it matches ALL rows for the given campaign. However,
`instructions.txt` always shows `voucher_name` as a required field in the action
template, so the LLM always fills it in — preventing a match-all update.

## Fix

Update `src/app/instructions.txt`:

1. **VOUCHER IDENTIFIER section** — add rule: if user says "all", "all tiers",
   "semua tier", or similar, omit `voucher_name` from the action block.

2. **PERFORMING UPDATES action template** — mark `voucher_name` as optional and
   add a note: omit it (do not include the key) when the request targets all tiers /
   all vouchers for a campaign.

## Tasks

### Task 1 — Update instructions.txt

**File:** `src/app/instructions.txt`

**Change A** — In "HOW TO HANDLE A USER REQUEST", section 3 (VOUCHER IDENTIFIER),
add a bullet:
```
- If the user says "all", "all tiers", "semua tier", or does not specify a tier → 
  target ALL vouchers for the campaign; omit `voucher_name` from the action block entirely
```

**Change B** — In "PERFORMING UPDATES", update the action template comment for
`voucher_name`:
- Change: `"voucher_name": "<exact Voucher Name from data>",`
- To: `"voucher_name": "<exact Voucher Name from data — OMIT this key entirely when targeting all tiers>",`

And add a note under IMPORTANT:
```
- When the user says "all tiers", "all vouchers", "semua tier", or omits the tier → 
  omit `voucher_name` from the action block entirely (do not include the key or set it to null).
  The system will then update ALL matching rows for the campaign.
```

## Verification

After the change, a prompt like "Update Campaign BAU Full Day all tier Min Spend to
RM1k" should produce an action block without `voucher_name`, which will update every
row with Campaign Name matching "BAU Full Day".
