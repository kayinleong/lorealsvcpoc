# Quick Task 260326-e71: Fix Field "undefined" not found in headers

**One-liner:** Added BulkChange interface and loop dispatch so LLM-generated changes arrays apply every row without hitting the "Field undefined not found in headers" error.

## What Was Done

The LLM correctly generates a `changes` array for bulk update requests (e.g. "Update all BAU Full Day Min Spend to RM1k"), but the `UpdateAction` interface only supported flat `field`/`value` fields. The handler passed `action` directly to `applyUpdate()`, which then tried to look up `action.field` — which was `undefined` — in the spreadsheet headers, causing the error.

Three changes were made to `src/app/app.ts`:

1. Added `BulkChange` interface (`campaign_name?`, `voucher_name?`, `field`, `value`).
2. Made `field` and `value` optional on `UpdateAction` and added `changes?: BulkChange[]`.
3. Replaced the single `applyUpdate(action)` call with a loop: if `action.changes` is present, iterate each entry and dispatch as a plain single-record `UpdateAction`; otherwise wrap the flat fields into a single-entry array and do the same — preserving backward compatibility.

`applyUpdate()` and `applyUpdateSurgical()` were not touched.
