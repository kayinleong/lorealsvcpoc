## Directory Purposes

**src/**
- Purpose: TypeScript source code for the bot application
- Contains: Entry point, configuration, and application logic
- Key files: `index.ts` (startup), `app/app.ts` (core logic), `config.ts` (settings)

**lib/**
- Purpose: Generated JavaScript output from TypeScript compilation
- Contains: Compiled `.js` and `.d.ts` files (declaration files)
- Generated: Yes - created by `npm run build` using tsc
- Committed: No - included in `.gitignore`

**appPackage/**
- Purpose: Microsoft Teams app manifest and visual assets
- Contains: `manifest.json` (app configuration), `color.png` and `outline.png` (icons)
- Generated: build/ subdirectory created during provisioning
- Committed: Manifest and icons committed, build/ not committed

**infra/**
- Purpose: Azure infrastructure as code for deployment
- Contains: Bicep templates for App Service, App Service Plan, Managed Identity, Bot Service
- Pattern: Modular - main `azure.bicep` orchestrates `botRegistration/azurebot.bicep`

**env/**
- Purpose: Environment-specific configuration files
- Contains: `.env.*` files with variable definitions for different environments
- Environments: dev, local, playground
- User overrides: `.env.*.user` files for local customizations

**devTools/**
- Purpose: Development utilities and tools
- Contains: M365 Agents Playground installation reference
- Generated: Playground is downloaded on first use

## Key File Locations

**Entry Points:**
- `src/index.ts`: Application startup - initializes app and starts HTTP server on port 3978
- `lib/src/index.js`: Compiled entry point (generated from `src/index.ts`)

**Configuration:**
- `src/config.ts`: Runtime configuration object - loads from environment variables
- `m365agents.yml`: Main lifecycle definition for Azure deployment (provision, deploy, publish)
- `m365agents.local.yml`: Local development lifecycle (local testing)
- `env/.env.*`: Environment variable files for different deployment targets
- `.localConfigs`: Generated runtime environment file for local development

**Core Logic:**
- `src/app/app.ts`: Bot application - message handlers, AI integration, storage
- `src/app/instructions.txt`: System prompt for AI agent behavior

**Deployment:**
- `infra/azure.bicep`: Azure resource definitions (App Service, identity, settings)
- `infra/botRegistration/azurebot.bicep`: Bot Framework registration (Bot Service, Teams channel)
- `infra/azure.parameters.json`: Bicep parameter values
- `appPackage/manifest.json`: Teams app manifest with bot configuration

**Manifests:**
- `package.json`: NPM dependencies and scripts
- `tsconfig.json`: TypeScript compilation settings
- `web.config`: IIS configuration for Azure App Service

## Naming Conventions

**Files:**
- TypeScript source: `camelCase.ts` (e.g., `app.ts`, `config.ts`, `instructions.txt`)
- Compiled output: Same name in `lib/` directory structure (e.g., `lib/src/app.js`)
- Configuration: `m365agents.{env}.yml` pattern for environment-specific lifecycles
- Environment files: `.env.{env}` and `.env.{env}.user` pattern

**Directories:**
- Source: `src/` with feature subdirectories (e.g., `src/app/`)
- Compiled: `lib/` mirrors `src/` structure
- Configuration: `env/` for environment files, `infra/` for deployment
- Package assets: `appPackage/` for Teams manifest and icons

**Variables (in config and env files):**
- camelCase: TypeScript source (`MicrosoftAppId`, `azureOpenAIKey`)
- SCREAMING_SNAKE_CASE: Environment variables and config files (`CLIENT_ID`, `AZURE_OPENAI_API_KEY`)

## Where to Add New Code

**New Feature (Message Handler or Handler Logic):**
- Primary code: `src/app/app.ts` - add new `app.on("event-name")` handler
- Example pattern: Line 58 for message handler, line 102 for feedback handler
- Storage: Use `storage.get()` and `storage.set()` with composite key pattern

**New Configuration Setting:**
- Definition: Add to `src/config.ts` object, loading from `process.env`
- Environment Files: Add variable to `env/.env.*` files
- Lifecycle: Update `m365agents.yml` to inject via `file/createOrUpdateEnvironmentFile` action

**New AI Behavior/Instructions:**
- Location: `src/app/instructions.txt` - plain text system prompt
- Access: Auto-loaded on startup via `loadInstructions()`
- Pattern: Modify text file directly; changes picked up on next startup

**New Azure Resource:**
- Location: `infra/azure.bicep` or new module in `infra/`
- Pattern: Define as resource block in Bicep, reference by output or parameter
- Export: Add output variable for subsequent actions to reference

**Utility Functions:**
- Location: Currently no `utils/` directory - add to `src/` if needed
- Pattern: Create `src/utils/` directory and export from main module as needed

**Error Handling or Logging:**
- Pattern: Add to try-catch block in message handler (`src/app/app.ts` line 63-99)
- Logging: Use `console.log()` and `console.error()`
